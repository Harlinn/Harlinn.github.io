Detailed Description

A critical section object provides synchronization similar to that provided by a mutex object, except that a critical section can be used only by the threads of a single process. Critical section objects cannot be shared across processes.

bool Harlinn::Common::Core::CriticalSection::try_lock ( ) const
inlinenoexcept

Attempts to enter a critical section without blocking. If the call is successful, the calling thread takes ownership of the critical section.

Returns

If the critical section is successfully entered or the current thread already owns the critical section, the return value is true.

If another thread already owns the critical section, the return value is false.
