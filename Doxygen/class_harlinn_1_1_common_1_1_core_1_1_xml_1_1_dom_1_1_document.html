NodeT Harlinn::Common::Core::Xml::Dom::Document::CreateNode ( const VARIANT & type,
BSTR name,
BSTR namespaceURI ) const

Creates a node using the supplied type, name, and namespace.

Template Parameters
NodeTThe interface wrapper type corresponding to the type argument.
Parameters
TypeA value that uniquely identifies the node type. This can be specified using either the integer value or the string value.
name
Parameters
namespaceURIThe namespace URI. If specified, the node is created in the context of the namespaceURI parameter with the prefix specified on the node name. If the name parameter does not have a prefix, this is treated as the default namespace.
Returns
A wrapper object for the new node.

The name parameter depends on the value of the Type parameter:

NodeType::Attribute The name of the attribute.
NodeType::CDataSection, NodeType::Comment, NodeType::Document, NodeType::DocumentFragment and NodeType::Text The nodeName property for these node types is a constant value; the name parameter is ignored.
NodeType::Element The name of the XML tag, with any namespace prefix included if present.
NodeType::Entity The name of the entity.
NodeType::EntityReference The name of the entity referenced. The name does not include the leading ampersand or the trailing semicolon. The name includes the namespace, if one is present.
NodeType::Notation The name of the notation.
NodeType::ProcessingInstruction The target, the first token following the <? characters.

You cannot create a node of type NodeType::Document, NodeType::DocumentType, NodeType::Entity, or NodeType::Notation.

When a node is created, it is created in the context of a namespace if one is supplied (if the namespaceURI parameter is supplied). If one is not supplied, the node is created in the namespace of the document. If the namespaceURI parameter is specified, the node is created in the context of the namespaceURI parameter with the prefix specified on the node name.

For node types that do not have names, the empty string, L"", should be passed as the name parameter.

For elements and entity references, when the value of the namespaceURI parameter is anything other than L"", and the value of the name parameter does not contain a prefix (xxx in xxx:yyy), the value of the namespaceURI parameter is treated as the default namespace.

Attributes cannot be scoped to a default namespace, and other elements are not qualified to a particular namespace; they are treated as being from the namespace defined by the document itself.

When the value of namespaceURI parameter is the empty string, L"", the node is created without a namespace. Creating a qualified node without specifying a nonempty namespaceURI returns an error. This member is an extension of the Worldwide Web Consortium (W3C) Document Object Model (DOM).
