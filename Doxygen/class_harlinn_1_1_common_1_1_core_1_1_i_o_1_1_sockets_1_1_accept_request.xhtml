Initializes a new AcceptRequest object.

Parameters
listenSocketA descriptor identifying a socket that has already been called with the listen function. A server application waits for attempts to connect on this socket.
acceptSocketA descriptor identifying a socket on which to accept an incoming connection. This socket must not be bound or connected.
bufferA pointer to a buffer that receives the first block of data sent on a new connection, the local address of the server, and the remote address of the client. The receive data is written to the first part of the buffer starting at offset zero, while the addresses are written to the latter part of the buffer. This parameter must be specified.
receiveDataLengthThe number of bytes in buffer that will be used for actual receive data at the beginning of the buffer. This size should not include the size of the local address of the server, nor the remote address of the client; they are appended to the output buffer. If receiveDataLength is zero, accepting the connection will not result in a receive operation. Instead, BeginAsyncAccept completes as soon as a connection arrives, without waiting for any data.
localAddressLengthThe number of bytes reserved for the local address information. This value must be at least 16 bytes more than the maximum address length for the transport protocol in use.
remoteAddressLengthThe number of bytes reserved for the remote address information. This value must be at least 16 bytes more than the maximum address length for the transport protocol in use. Cannot be zero.

The easiest way to get the size of the allocation for buffer right is to use the static CalculateBufferSizeFor function to calculate the required size. and then just accept the default values for localAddressLength and remoteAddressLength.

static constexpr UInt32 Harlinn::Common::Core::IO::Sockets::AcceptRequest::CalculateBufferSizeFor ( UInt32 receiveDataLength)
inlinestaticconstexpr

Calculates the allocation size, in bytes, for the buffer parameter to the constructors of this class. This is required because this buffer will receive:

  1. The first block of data received from the client.
  2. The remote addresses for the connection.
  3. The local addresses for the connection.
Parameters
receiveDataLengthThe expected number of bytes for the first block of data received from the client.
Returns
The number of bytes that must be allocated for the buffer parameter to the constructors to be able to receive receiveDataLength number of bytes while processing the asynchronous request initiated by a call to BeginAsyncAccept.
