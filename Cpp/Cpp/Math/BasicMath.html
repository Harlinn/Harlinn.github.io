Optimized basic math functions

C++ is often used to create software that performs mathematical operations efficiently.

Complex computations are put together using the functionality provided by the standard C and C++ libraries as the basic building blocks. The header, HCCMath.h provides alternatives to many of the basic mathematical function that is specified in the C and C++ standards. All of the functions can be constexpr evaluated, and several offer surprising performance benefits as well.

The functions are implemented in the Harlinn::Common::Core::Math namespace.

The performance the functions are thoroughly checked by a test program included in the Harlinn.Windows solution.

Each function, together with its C/C++ counterpart, is executed 1 000 000 000 times, to check if there are any performance benefits to using the constexpr implementations at runtime, and that they return the same results for the same arguments.

Basic operations

  • Abs which returns the absolute value |x| for x, is implemented for floating point types, signed integers and unsigned integers.
  • fmod which calculate the remainder of a floating point division operation, is implemented for floating point types.
  • Max which returns the greater of to values, is implemented for floating point types.
  • Min which returns the lesser of to values, is implemented for floating point types.
  • IsSameValue checks for binary equality between two floating point values.

Exponential functions

  • Exp returns e raised to the given power (ex).
  • Log computes natural, base e, logarithm (ln x)
  • Log2 base 2 logarithm of the given number (log2x). When applied to single precision floating point values, the implementation is 251.54 % faster than the standard implementation.
  • Log10 computes common (base 10) logarithm (log10x )

Power functions

  • Sqrt computes square root (√x)
  • Hypot computes square root of the sum of the squares of two numbers

Trigonometric functions

Graphic intensive application are highly sensitive to the performance of the trigonometric functions, especially for single precision floating point values.

  • Sin computes the sine of its argument given in radians.

    When applied to single precision floating point values, the implementation is 14.38 % faster than the standard implementation.

  • Cos computes the cosine of its argument given in radians.

    When applied to single precision floating point values, the implementation is 14.93 % faster than the standard implementation.

  • Tan computes the tangent of its argument given in radians.

  • ASin computes arc sine of its argument.

    When applied to double precision floating point values, the implementation is 59.78 % faster than the standard implementation, and similarly 109.87 % faster for single precision floating point values.

  • ACos computes the arc cosine of its argument.

    When applied to double precision floating point values, the implementation is 90.89 % faster than the standard implementation, and similarly 107.69 % faster for single precision floating point values.

  • ATan computes the arc tangent of its argument.

    When applied to double precision floating point values, the implementation is 6.56 % faster than the standard implementation, and similarly 78.08 % faster for single precision floating point values.

  • ATan2 computes the arc tangent of x / y, its two arguments, using signs to determine quadrants.

    When applied to double precision floating point values, the implementation is 44.03 % faster than the standard implementation, and similarly 36.17 % faster for single precision floating point values.

Nearest integral value floating point operations

  • Ceil computes the nearest integral value not less than the given value.

    When applied to double precision floating point values, the implementation is 20.19 % faster than the standard implementation, and similarly 19.98 % faster for single precision floating point values.

  • Floor computes the nearest integral value not greater than the given value.

    When applied to double precision floating point values, the implementation is 18.18 % faster than the standard implementation, and similarly 20.50 % faster for single precision floating point values.

  • Trunc computes the nearest integral value not greater in magnitude than the given value.

    When applied to double precision floating point values, the implementation is 724.39 % faster than the standard implementation, and similarly 6.54 % faster for single precision floating point values.

  • Round computes the nearest integral value, rounding away from zero in halfway cases.

    When applied to double precision floating point values, the implementation is 1077.61 % faster than the standard implementation, and similarly 38.27 % faster for single precision floating point values.

Floating point manipulation functions

  • FRExp decomposes a number into significand and base-2 exponent.

    When applied to double precision floating point values, the implementation is 587.58 % faster than the standard implementation, and similarly 622.41 % faster for single precision floating point values.

  • ModF decomposes a number into integer and fractional parts.

    When applied to double precision floating point values, the implementation is 91.48 % faster than the standard implementation, and similarly 49.41 % faster for single precision floating point values.

  • ScaleByN multiplies a number by FLT_RADIX raised to a power.

    When applied to double precision floating point values, the implementation is 421.72 % faster than the standard implementation, and similarly 467.42 % faster for single precision floating point values.

  • NextAfter next representable floating-point value towards the given value.

    When applied to double precision floating point values, the implementation is 740.07 % faster than the standard implementation, and similarly 232.39 % faster for single precision floating point values.

  • NextUp Return the smallest floating point number y of the same type as x such that x < y. If no such y exists, e.g. if x is Inf or NaN, then return x.

    The standard C++ implementation is std::nextafter( x, std::numeric_limits<double>::infinity( ) ).

    When applied to double precision floating point values, the implementation is 2783.78 % faster than the standard implementation, and similarly 586.95 % faster for single precision floating point values.

  • NextDown Return the largest floating point number y of the same type as x such that y < x. If no such y exists, e.g. if x is -Inf or NaN, then return x.

    The standard C++ implementation is std::nextafter( x, -std::numeric_limits<double>::infinity( ) ).

    When applied to double precision floating point values, the implementation is 1839.04 % faster than the standard implementation, and similarly 370.61 % faster for single precision floating point values.

  • CopySign copies the sign of a floating point value.

    When applied to double precision floating point values, the implementation is 493.48 % faster than the standard implementation, and similarly 1.82 % faster for single precision floating point values.

Classification and comparison

  • IsNaN checks if the given number is NaN.

    When applied to double precision floating point values, the implementation is 5.51 % faster than the standard implementation, and similarly 6.62 % faster for single precision floating point values.

  • IsInf checks if the given number is infinite.

    When applied to double precision floating point values, the implementation is 3.83 % faster than the standard implementation, and similarly 4.07 % faster for single precision floating point values.

  • SignBit checks if the given number is negative.

    When applied to double precision floating point values, the implementation is 82.79 % faster than the standard implementation, and similarly 101.32 % faster for single precision floating point values.

Other computations

  • Clamp If the a value is within [minimumValue, maximumValue], the function returns the value, otherwise it returns the nearest boundary.

    When applied to double precision floating point values, the implementation is 3.57 % faster than the standard implementation, and similarly 9.15 % faster for single precision floating point values.

  • Lerp Computes the linear interpolation between a and b, if the parameter t is inside [​0​, 1), the linear extrapolation otherwise, i.e. the result of a + t * ( b - a ) with accounting for floating point calculation imprecision.

    When applied to double precision floating point values, the implementation is 33.34 % faster than the standard implementation, and similarly 38.38 % faster for single precision floating point values.
